## 1. Community Strategy Overview

### Community Purpose

Finance's community exists to:

1. **Support users** — Help people get the most from Finance
2. **Collect feedback** — Direct line from users to the development team
3. **Build trust** — Transparent development builds confidence in a privacy-first product
4. **Share knowledge** — Financial wellness tips and budgeting strategies (not financial advice)

### Community Values

| Value                  | In Practice                                                                            |
| ---------------------- | -------------------------------------------------------------------------------------- |
| **Inclusive**          | All financial situations welcome. No assumptions about income, literacy, or ability.   |
| **Privacy-respecting** | Never ask users to share financial data. No screenshots of real financial information. |
| **Non-judgmental**     | No comments about spending habits. No "you should have..." responses.                  |
| **Helpful**            | Answer questions directly. Share what works for you without prescribing.               |
| **Transparent**        | Development decisions are explained. Roadmap is public. Limitations are acknowledged.  |

### What the Community is NOT

- ❌ Not a financial advice forum (we're a tool, not advisors)
- ❌ Not a customer support replacement (bugs go to GitHub Issues)
- ❌ Not a marketing channel (community value first, promotion never)
- ❌ Not a data collection tool (we don't mine community posts for analytics)

## 3. Community Guidelines

_(Aligned with existing CODE_OF_CONDUCT.md and extended for Finance-specific contexts)_

### Welcome to the Finance Community

Finance is a space for people who want to track their finances privately and thoughtfully. These guidelines help keep the community welcoming for everyone.

### The Basics

1. **Be kind and respectful.** Everyone here is at a different point in their financial journey. Meet people where they are.

2. **No financial advice.** Share what works for _you_, but don't tell others what they _should_ do with their money. "I found envelope budgeting helpful for groceries" is great. "You need to stop eating out" is not.

3. **Protect your privacy (and others').** Never share real financial data — yours or anyone else's. Use sample data in screenshots. Don't ask others to share their balances, income, or spending.

4. **No judgment about spending.** This community does not evaluate how people spend their money. No comments about purchases, priorities, or lifestyle choices.

5. **Keep it constructive.** Feedback on Finance (positive or critical) is always welcome. "This feature doesn't work well because..." is helpful. "This app sucks" isn't.

6. **No self-promotion or spam.** Don't post links to competing products, affiliate links, or promotional content. Genuinely helpful tool recommendations in context of a discussion are fine.

7. **Respect the team's time.** The Finance team reads every post, but responses may take 24–48 hours. Don't cross-post the same question in multiple categories.

### Specifically Not Allowed

- Sharing or requesting real financial data (account numbers, balances, transaction details)
- Financial advice, recommendations, or prescriptions
- Judgment about spending habits, income level, or financial decisions
- Harassment, discrimination, or personal attacks (see CODE_OF_CONDUCT.md)
- Commercial spam or self-promotion
- Requests for features or fixes with aggressive/demanding language

### Reporting Issues

If you see a post that violates these guidelines, use GitHub's report feature or email [moderation email]. We review reports within 24 hours.

## 5. Moderation Strategy

### Moderation Roles

| Role                  | Responsibilities                                     | Who              |
| --------------------- | ---------------------------------------------------- | ---------------- |
| **Primary moderator** | Daily review, respond to reports, enforce guidelines | [Human — assign] |
| **Backup moderator**  | Cover when primary is unavailable                    | [Human — assign] |
| **Community manager** | Welcome new members, seed discussions, maintain tone | [Human — assign] |

### Response SLA

| Report Type                 | Response Time             |
| --------------------------- | ------------------------- |
| Harassment / safety concern | 4 hours                   |
| Spam                        | 24 hours                  |
| Guideline violation         | 24 hours                  |
| Feature request / question  | 48 hours (acknowledgment) |

### Moderation Actions

| Violation                        | First Offense                          | Repeat        | Severe        |
| -------------------------------- | -------------------------------------- | ------------- | ------------- |
| Off-topic / low effort           | Gentle redirect                        | Lock thread   | —             |
| Self-promotion / spam            | Remove post, friendly note             | Temporary ban | Permanent ban |
| Financial advice                 | Gentle correction + link to guidelines | Warning       | —             |
| Sharing real financial data      | Remove immediately, DM explanation     | Warning       | —             |
| Harassment / discrimination      | Remove + warning                       | 7-day ban     | Permanent ban |
| Privacy violation (others' data) | Remove immediately                     | Permanent ban | —             |

### Response Templates

**Welcome (for new members):**

> Welcome to the Finance community! 👋 We're glad you're here. Feel free to introduce yourself in General — and check out Tips & Guides for community budgeting strategies.

**Redirect (off-topic):**

> Thanks for the post! This might fit better in [other category]. Would you mind moving it there? It'll get more visibility from the right people.

**Financial advice correction:**

> We appreciate you sharing your experience! Just a friendly reminder: we keep advice personal ("this worked for me") rather than prescriptive ("you should do X"). Everyone's situation is different. 🙂

**Privacy reminder:**

> Hey! It looks like this screenshot might contain real financial data. For your privacy, we've removed it. Feel free to repost with sample data instead. We want to protect your information.

## 6. Community Growth Plan

### Phase 1: Foundation (Weeks 1–4)

- [ ] GitHub Discussions enabled with all categories
- [ ] Templates configured for Feature Requests and Bug Reports
- [ ] Community guidelines pinned in General
- [ ] Welcome post from team in Announcements
- [ ] Social media accounts created with consistent branding
- [ ] Pinned post on Twitter/X
- [ ] Team members active in discussions (responding within 48 hours)

### Phase 2: Beta Community (Weeks 5–8)

- [ ] Beta testers directed to Discussions for feedback
- [ ] Weekly engagement posts (polls, "what are you testing?" threads)
- [ ] Feature request discussions actively responded to
- [ ] Cross-post blog content summaries to relevant categories
- [ ] Share beta milestone updates in Announcements

### Phase 3: Pre-Launch Growth (Weeks 9–12)

- [ ] Community-contributed Tips & Guides content appearing
- [ ] At least 3 community members regularly helping others
- [ ] Feature request prioritization visible (respond with "planned" / "considering")
- [ ] Launch countdown in Announcements
- [ ] Community members invited to be launch day advocates (opt-in)

### Phase 4: Post-Launch (Ongoing)

- [ ] New user welcome flow (auto-message or pinned getting-started guide)
- [ ] Weekly "What's New" updates in Announcements
- [ ] Monthly community roundup (top discussions, feature requests, stats)
- [ ] Community contributors recognized (without gamification pressure)

### Quantitative Metrics

| ------------------------------ | ------- | ---------------- | ---------------- |
| GitHub Discussions posts       | GitHub  | 30+              | 100+             |
| Active discussion participants | GitHub  | 15+              | 50+              |
| Feature requests submitted     | GitHub  | 10+              | 30+              |
| Bug reports submitted          | GitHub  | 20+              | 40+              |
| Twitter/X followers            | Twitter | 200+             | 500+             |
| GitHub stars                   | GitHub  | 50+              | 200+             |

### Qualitative Health Indicators

| Healthy Signal                       | Unhealthy Signal                  |
| ------------------------------------ | --------------------------------- |
| New members introducing themselves   | Crickets — no one posts           |
| Community members helping each other | Only team responding to questions |
| Constructive feature discussions     | Demanding or entitled requests    |
| Diverse perspectives and use cases   | Echo chamber of one type of user  |
| Honest critical feedback             | Only positive or only negative    |
| Questions answered within 48 hours   | Unanswered questions piling up    |

### Red Flags to Watch For

- Moderation reports increasing week-over-week
- Community members sharing real financial data
- Financial advice being given (potential liability)
- Negative sentiment dominating without constructive framing
- Spam or astroturfing from competitors
